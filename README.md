# E-Commerce Customer Segmentation (RFM Analytics)

## Overview
This end-to-end retail analytics project utilizes Python for data cleaning, PostgreSQL for RFM-based customer segmentation, and Power BI for interactive dashboards. It analyzes customer behavior, revenue trends, and segment performance on the Online Retail II dataset to generate business insights and data-driven recommendations.

Customer segments identified include:
- **Champion**: Highly valuable customers who purchase frequently and recently.
- **Loyal**: Consistently good customers.
- **Promising**: Recent shoppers but haven't spent much yet.
- **At Risk**: Former good customers who haven't purchased recently.
- **Cannot Lose**: High-value customers who haven't returned in a long time.
- **Lost**: Customers who have churned.
- **Needs Attention**: Customers whose behavior indicates they might churn soon.

## Project Structure
- **`code/`**: Contains the main Python scripts/Jupyter Notebooks for data preprocessing and RFM scoring (`rfm_analysis.ipynb` and `01_rfm_score.sql`).
- **`sql_queries/`**: Contains SQL scripts used to extract segmented insights (e.g., segment analysis, product affinity, basket analysis).
- **`csv_reports/` & `sql_results/`**: Extracted CSV and Excel reports based on the SQL insights (monthly revenue, at-risk by country, champion vs lost, segment insights, etc.).
- **`dashboards/`**: The PowerBI `.pbix` file which connects to the processed data for interactive visualization.
- **`dataset/` & `datasets_cleaned/`**: Source and cleaned datasets for the project.

## Key Findings
- **Revenue by Segment**: Analyzed the proportion of revenue generated by each customer segment.
- **At-Risk by Country**: Highlighted geographies with the highest concentration of "At Risk" and "Lost" customers.
- **Product Affinity & Basket Analysis**: Explored which products are frequently bought together to drive upselling and cross-selling campaigns.

## How to Use
1. **Data Processing**: Run the notebooks in the `code/` directory to generate RFM scores and segments.
2. **SQL Analysis**: Use the scripts in `sql_queries/` on your SQL database to generate aggregated CSV reports.
3. **Visualization**: Open `dashboards/online_retail.pbix` in PowerBI Desktop to view the interactive dashboard. (Ensure data paths in PowerBI point to the correct CSV outputs).

## Technologies Used
- **Python (Pandas, Jupyter)** for Data Wrangling and RFM computation.
- **PostgreSQL** for deeper aggregations and segment comparisons.
- **Power BI** for creating an interactive analytics dashboard.
